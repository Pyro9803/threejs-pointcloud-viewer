# 3D Tiles Point Cloud Viewer

A browser-based viewer for 3D Tiles point cloud tilesets generated by
[py3dtiles](https://py3dtiles.org/) from LAS/LAZ data. Built with
[Three.js](https://threejs.org/) and NASA AMMOS
[3d-tiles-renderer](https://github.com/NASA-AMMOS/3DTilesRendererJS).

## Quick start

```bash
cd threejs-pointcloud-viewer
npm install
npm run dev
```

Open `http://localhost:5173` in your browser.

## Adding your tileset

1. Generate a tileset with py3dtiles:

   ```bash
   py3dtiles convert --out output/ input.las
   ```

2. Copy the entire output directory into `public/tileset/`:

   ```
   public/
   └── tileset/
       ├── tileset.json
       └── *.pnts  (or subdirectories of tiles)
   ```

3. Refresh the browser — the viewer loads `/tileset/tileset.json` automatically.

## Why `file://` doesn't work (CORS)

Browsers enforce the
[Same-Origin Policy](https://developer.mozilla.org/en-US/docs/Web/Security/Same-origin_policy)
for `fetch()` requests. When you open `index.html` directly from the
filesystem (`file://`), every request to load a tile is treated as a
cross-origin request and blocked.

The Vite dev server (`npm run dev`) serves everything over `http://localhost`
so all requests share the same origin. Always use the dev server.

## py3dtiles ECEF coordinates

py3dtiles reprojects LAS point clouds into
[ECEF (Earth-Centered, Earth-Fixed)](https://en.wikipedia.org/wiki/Earth-centered,_Earth-fixed_coordinate_system)
coordinates by default. In ECEF the origin is the Earth's centre, so
bounding-sphere centres have magnitudes around 6 371 000 m (Earth's radius).

The viewer detects this automatically:

- Camera is placed along the local surface-normal direction so the cloud
  appears right-side-up.
- The info panel converts the ECEF centre to approximate latitude/longitude
  for display.
- `camera.far` is set to 100× the bounding sphere radius so no tiles are
  clipped.

If your tileset uses a local/projected CRS instead, the viewer falls back to
standard XYZ positioning.

## UI controls

| Control | Description |
|---|---|
| **Point size** slider | Adjusts `PointsMaterial.size` live (0.01 – 0.5) |
| **Toggle background** | Switches between dark and light background |
| **FPS counter** | Top-right corner, updated every 500 ms |
| **Loading indicator** | Bottom-centre, shows `loaded / total` tile count |
| **Tileset info** | Bounding sphere centre (lat/lon or XYZ) and radius |

## Build for production

```bash
npm run build   # outputs to dist/
npm run preview # local preview of the production build
```

Copy `dist/` to any static web host. Remember to also copy `public/tileset/`
next to it (or configure your host to serve the tiles from the same origin).
